## 安全边界与禁止事项

### 必须遵守

- 先确认 `agent_id`，再执行诊断
- 优先 `list_agents` → `agent_info` → `arthas_status`
- 已有专用 MCP tool 时，不要降级为 `arthas_exec`
- 需要等待触发的命令优先使用异步 session
- 会话结束后必须 `arthas_session_close`
- 整轮诊断结束后优先考虑 `arthas_detach`

### 不要这样做

- 不要在没有事实依据时假设目标实例
- 不要把外部 Arthas 文档里的所有 flag 直接映射到当前 MCP 参数
- 不要在一个回答里同时发起多个难以收尾的长命令
- 不要把“无结果”直接等价为“无问题”
- 不要把推断写成确定结论
- 不要遗留长期存活的 session 或 Arthas 进程

### 推荐的表述方式

- **事实**：来自工具返回
- **推断**：基于事实的分析
- **后续动作**：明确的下一步操作

### 推荐的执行姿势

- 先做低风险发现，再做针对性观察
- 先做一次性验证，再决定是否进入长时观察
- 尽量缩小类、方法、表达式范围
- 长观察一定要可中断、可 pull、可关闭
